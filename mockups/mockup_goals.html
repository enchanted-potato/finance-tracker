Implementation Notes
Data Model Needed
  • goals table: id, user_id, name, account_type_id (optional link), target_amount, target_date
  • Current value = latest snapshot balance for linked account
  • Status = computed: compare progress rate vs required rate to hit target
  • No separate GoalType needed — free text or fixed categories
Status Logic
  • Ahead — actual % > expected % by >5pp
  • On Track — within ±5pp of expected %
  • Behind — actual % < expected % by >5pp
  • Expected % = (elapsed time / total duration) × 100
