   - Return the decoded dict on success (has `uid`, `email`, `name` or `display_name` field â€” note: the field is `name` in Firebase decoded token).

   - Query User by id=uid. If exists, return it.
   - If not, create User(id=uid, email=email, display_name=display_name), add, commit, return.
   - Import User from app.models, Session from sqlmodel.

   Do NOT import streamlit anywhere in this file â€” services must be Streamlit-independent per project convention.

<task type="auto">
  <name>Task 2: Firebase auth custom Streamlit component</name>
  <files>frontend/auth_component/index.html</files>
  <action>
Create `frontend/auth_component/index.html` â€” a single-file custom Streamlit component (no build step, no npm).

The component must:

1. **Load Firebase JS SDK 12.9.0** from gstatic.com CDN (ES module imports):
   - `https://www.gstatic.com/firebasejs/12.9.0/firebase-app.js` (initializeApp)
   - `https://www.gstatic.com/firebasejs/12.9.0/firebase-auth.js` (getAuth, signInWithPopup, signOut, GoogleAuthProvider, onAuthStateChanged, browserLocalPersistence, setPersistence)

2. **Implement the Streamlit component protocol:**
   - On load, send `{ isStreamlitMessage: true, type: "streamlit:componentReady" }` to `window.parent`.
   - Listen for `"streamlit:render"` messages from parent to receive `args` (contains `firebase_config` dict and optionally `action`).
   - When sending values back, use:
     ```javascript
     window.parent.postMessage({
       isStreamlitMessage: true,
       type: "streamlit:setComponentValue",
       value: theValue,
       dataType: "json",
     }, "*");
     ```
   - After setting value, send `{ isStreamlitMessage: true, type: "streamlit:setFrameHeight", height: N }` to control iframe size.

3. **Three-state protocol (sent as component value):**
   - `{ "status": "initializing" }` â€” sent immediately before onAuthStateChanged fires.
   - `{ "status": "authenticated", "token": "<idToken>" }` â€” sent when user is signed in. Get fresh token with `user.getIdToken()`.
   - `{ "status": "unauthenticated" }` â€” sent when no session exists.

4. **UI rendering:**
   - When unauthenticated: render a centered branded login card with:
     - App name: "Net Worth Tracker"
     - Subtitle: "Track your net worth over time"
     - A "Sign in with Google" button (styled cleanly â€” white background, Google colors, border radius).
     - Set iframe height to 500.
   - When authenticated or initializing: hide all UI, set iframe height to 0.
   - On error (popup blocked, auth failure): show error message below login button, keep the button visible so user can retry.

5. **Logout support:** Listen for `args.action === "logout"` in render messages. When received, call `signOut(auth)` â€” the `onAuthStateChanged` listener will fire and send `"unauthenticated"` status automatically.

6. **Persistence:** Call `setPersistence(auth, browserLocalPersistence)` before setting up `onAuthStateChanged`. This enables transparent re-auth on page reload.

Important implementation notes:
- Use ES modules with `<script type="module">` â€” Firebase 12.x is ESM-only from gstatic.
- The component protocol requires sending `componentReady` BEFORE any render message arrives.
- Do NOT use `Streamlit.setComponentValue()` JS helper â€” there is no helper available in zero-build components. Use raw `postMessage` directly.
- Only call `signInWithPopup` inside a user click handler (browser requirement for popups).
- Call `user.getIdToken()` to get fresh token on every auth state change, not `user.accessToken`.
  </action>
