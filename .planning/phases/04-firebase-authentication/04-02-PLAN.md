<task type="auto">
  <name>Task 1: Integrate auth gate into main.py</name>
  <files>frontend/main.py</files>
  <action>
Rewrite `frontend/main.py` to replace the hardcoded TEST_USER_ID with real Firebase auth. The overall structure:

1. **Remove all test user code:** Delete `TEST_USER_ID`, `TEST_USER_EMAIL`, `TEST_USER_NAME`, `_ensure_test_user()`. The user auto-creation now happens through `get_or_create_user` in auth_service.

2. **Declare the custom component** at module level:
   ```python
   import streamlit.components.v1 as components
   import os

   _AUTH_COMPONENT_DIR = os.path.join(os.path.dirname(__file__), "auth_component")
   _auth_component = components.declare_component("firebase_auth", path=_AUTH_COMPONENT_DIR)
   ```

3. **Create a helper function** to call the component:
   ```python
   def _firebase_auth_widget(firebase_config: dict, action: str = "") -> dict | None:
       return _auth_component(firebase_config=firebase_config, action=action, default=None)
   ```

4. **Create `_auth_gate()` function** — called early in `main()`, before any page content:

   a. If `st.session_state.get("user_id")` is already set, return immediately (session persists across Streamlit reruns — this is how navigation works without re-login).

   b. Build `firebase_config` dict from settings: `{ "apiKey": settings.firebase_web_api_key, "authDomain": settings.firebase_auth_domain, "projectId": settings.firebase_project_id }`.

   c. Check if user clicked logout (use a session_state flag `_logout_requested`). If so, call the widget with `action="logout"`, clear session_state keys (`user_id`, `user_email`, `user_name`, `_logout_requested`), and call `st.rerun()`.

   d. Call the widget: `result = _firebase_auth_widget(firebase_config)`.

   e. Handle the three states:
      - `result is None` (first render): Show nothing (blank). Call `st.stop()`.
      - `result["status"] == "initializing"`: Show a centered `st.spinner("Loading...")`. Call `st.stop()`.
      - `result["status"] == "unauthenticated"`: The component renders the login UI itself (iframe shows the card). Hide sidebar with `st.markdown("<style>[data-testid='stSidebar']{display:none}</style>", unsafe_allow_html=True)`. Call `st.stop()`.
      - `result["status"] == "authenticated"`: Extract `result["token"]`. Call `verify_firebase_token(token)`. If valid, extract `uid`, `email`, `name` (use `.get("name", "")` since name may be absent). Call `get_or_create_user(session, uid, email, name)`. Set `st.session_state["user_id"] = uid`, `st.session_state["user_email"] = email`, `st.session_state["user_name"] = name`. Call `st.rerun()` to proceed past the gate. If token invalid, show `st.error("Authentication failed. Please try again.")` and `st.stop()`.

5. **Update `_init_db()`:** Remove `_ensure_test_user(session)` call. Add `init_firebase_admin()` call (import from auth_service).

6. **Update `main()` flow:**
   - Keep `set_page_config` and custom CSS at the top.
   - Keep the DB init block.
   - Call `_auth_gate()` immediately after DB init — if user is not authenticated, `st.stop()` prevents anything below from rendering.
   - After the gate: sidebar shows navigation as before.
   - Update sidebar header: replace `TEST_USER_NAME` with `st.session_state.get("user_name", st.session_state.get("user_email", ""))`. Show display name if available, fall back to email.
   - Add logout button at bottom of sidebar (below the nav loop, after a divider): `if st.sidebar.button("🚪  Log out", key="nav_logout", use_container_width=True): st.session_state["_logout_requested"] = True; st.rerun()`.
   - Keep the page rendering match statement as-is.

7. **Important details:**
   - The `st.session_state["user_id"]` key is what all pages already read via `_get_user_id()` — no changes needed to any page files.
   - Use a `get_session()` context for the `get_or_create_user` call in the auth gate.
   - The CSS to hide the sidebar must only be injected when unauthenticated. When authenticated, the sidebar shows normally.
   - For the logout flow: setting `_logout_requested` flag and rerunning ensures the component receives the `action="logout"` on the next render cycle, which triggers `signOut` in the JS, which triggers `onAuthStateChanged` → unauthenticated state.
  </action>

  Verify auth gate exists: `grep "auth_gate\|declare_component\|verify_firebase_token\|get_or_create_user" frontend/main.py` — should show matches.
  </verify>

The script should:

1. Accept the target Firebase UID as a command-line argument: `python scripts/migrate_test_user.py <firebase_uid>`.
2. Use `argparse` with a required positional arg for the UID.
3. Connect to the database using the same `engine` from `app.database`.
4. Run these SQL statements in a single transaction (use `sqlmodel.Session` or raw `connection.execute`):
   ```sql
   UPDATE account_types SET user_id = :new_uid WHERE user_id = 'test-user';
   UPDATE accounts SET user_id = :new_uid WHERE user_id = 'test-user';
   UPDATE liabilities SET user_id = :new_uid WHERE user_id = 'test-user';
   UPDATE snapshots SET user_id = :new_uid WHERE user_id = 'test-user';
   DELETE FROM users WHERE id = 'test-user';
   ```
5. Print the number of rows affected for each table.
6. Add a confirmation prompt before executing: "This will migrate all data from 'test-user' to '{uid}'. Continue? [y/N]".
7. Add a `--dry-run` flag that prints the SQL without executing.

Include a docstring at the top explaining what this does and when to run it (Phase 5, after first real login).
