<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Firebase Auth | signInWithPopup crosses the boundary; Firebase handles token issuance |
| Browser → FastAPI | Every API request carries a Firebase ID token as Bearer header |
| Build artefact | .env (containing real Firebase keys) must never be committed or included in build output |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Information Disclosure | Firebase ID token | mitigate | Token never stored in localStorage, React state, or any persisted storage. Axios interceptor calls `getIdToken(user)` per request and discards the string immediately. |
| T-11-02 | Elevation of Privilege | Stale auth state | mitigate | PrivateRoute checks `loading` before `user`; loading=true renders null (no flash). 401 response interceptor calls `window.location.href = '/login'` — hard redirect clears React state. |
| T-11-03 | Information Disclosure | .env secrets in git | mitigate | client/.gitignore includes `.env`. `.env.example` committed with placeholder values only. Real keys sourced from Firebase console at deploy time. |
| T-11-04 | Tampering | CSRF via cross-origin state mutation | accept | All state changes go through FastAPI with Bearer token validation. CORS allow-list in api/main.py restricts origins to localhost:5173 and Firebase Hosting URLs — already in place from Phase 9. |
| T-11-05 | Denial of Service | Google Sign-In popup blocked by browser | accept | Browser popup-blockers can block `signInWithPopup`. This is a known UX degradation, not a security threat. `signInWithRedirect` fallback deferred to a later phase; document in error handler comment in LoginPage.tsx. |
| T-11-06 | Information Disclosure | XSS token extraction | mitigate | Token only exists as a transient local variable inside the Axios interceptor function scope — never written to DOM, localStorage, or sessionStorage. XSS would need to intercept an in-flight request, which is not feasible in modern browsers with CSP headers (to be added in Phase 15). |
</threat_model>
