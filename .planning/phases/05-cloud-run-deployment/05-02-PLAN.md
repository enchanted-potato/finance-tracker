Create Terraform configuration to automate GCP infrastructure provisioning.

**Create terraform/ directory structure:**

**1. terraform/main.tf** - Core infrastructure:
```hcl
terraform {
  required_providers {
    google = {
      source  = "hashicorp/google"
      version = "~> 5.0"
    }
  }
}

provider "google" {
  project = var.project_id
  region  = var.region
}

# Cloud SQL instance
resource "google_sql_database_instance" "main" {
  name             = "finance-tracker-db"
  database_version = "POSTGRES_15"
  region           = var.region
  deletion_protection = false

  settings {
    tier              = "db-f1-micro"  # Free tier
    availability_type = "ZONAL"
    disk_size         = 10
    disk_type         = "PD_HDD"

    database_flags {
      name  = "cloudsql.iam_authentication"
      value = "on"
    }

    backup_configuration {
      enabled = false  # Disabled for free tier
    }
  }
}

# Database
resource "google_sql_database" "database" {
  name     = "finance_tracker"
  instance = google_sql_database_instance.main.name
}

# IAM database user for Cloud Run service account
resource "google_sql_user" "iam_user" {
  name     = data.google_compute_default_service_account.default.email
  instance = google_sql_database_instance.main.name
  type     = "CLOUD_IAM_SERVICE_ACCOUNT"
}

# Data source for default compute service account
data "google_compute_default_service_account" "default" {}
```

<task type="auto">
  <name>Initialize database schema on Cloud SQL</name>
  <files>N/A - infrastructure task</files>
  <action>
Create all database tables in Cloud SQL production instance using the existing `main.py` schema creation script. This must happen BEFORE data migration in Plan 04.

**Approach:** Use Cloud SQL Proxy to create a local Unix socket connection, then run `python main.py` which calls `SQLModel.metadata.create_all(engine)`.

**Steps:**

1. **Download and install Cloud SQL Proxy** (if not already installed):
```bash
curl -o cloud-sql-proxy https://storage.googleapis.com/cloud-sql-connectors/cloud-sql-proxy/v2.21.1/cloud-sql-proxy.darwin.amd64
chmod +x cloud-sql-proxy
mv cloud-sql-proxy /usr/local/bin/
```

2. **Start Cloud SQL Proxy in background:**
```bash
cloud-sql-proxy {INSTANCE_CONNECTION_NAME} &
PROXY_PID=$!
```

This creates a Unix socket at `/cloudsql/{INSTANCE_CONNECTION_NAME}`.

3. **Set DATABASE_URL to use Unix socket:**
```bash
export DATABASE_URL="postgresql+psycopg2://{SERVICE_ACCOUNT_EMAIL}:@/finance_tracker?host=/cloudsql/{INSTANCE_CONNECTION_NAME}"
```

Note: Password is empty (`:@`) for IAM authentication. IAM auth with Cloud SQL Proxy uses service account credentials from `gcloud auth application-default login`.

4. **Run schema creation:**
```bash
python main.py
```

This executes `SQLModel.metadata.create_all(engine)` which creates all tables (users, accounts, liabilities, account_types, liability_types, snapshots).

5. **Verify tables created:**
```bash
# Connect via proxy and list tables
gcloud sql connect finance-tracker-db --user={SERVICE_ACCOUNT_EMAIL} --database=finance_tracker -e "\dt"
```

Expected output: Tables list including users, accounts, liabilities, account_types, liability_types, snapshots.

6. **Stop Cloud SQL Proxy:**
```bash
kill $PROXY_PID
```

**If IAM auth fails:** Fall back to database password approach:
- Retrieve password from Cloud SQL instance creation
- Update DATABASE_URL: `postgresql+psycopg2://postgres:{PASSWORD}@/finance_tracker?host=/cloudsql/{INSTANCE_CONNECTION_NAME}`
- Re-run main.py

**Why this task is in Plan 02:** Schema tables must exist BEFORE Plan 04 (data migration) can run. Migration script updates existing rows, which requires tables to exist. Per CONTEXT.md, migration happens before deployment, so schema initialization must happen before migration.
  </action>
  <verify>gcloud sql connect finance-tracker-db --user={SERVICE_ACCOUNT_EMAIL} --database=finance_tracker -e "\dt" | grep -E "(users|accounts|liabilities|snapshots)"</verify>
  <done>All tables (users, accounts, liabilities, account_types, liability_types, snapshots) exist in Cloud SQL production database, ready for data migration</done>
</task>
