<task type="auto">
  <name>Run database schema migration on Cloud SQL</name>
  <files>N/A - infrastructure task</files>
  <action>
Create all database tables in Cloud SQL production instance before deployment using the existing `main.py` schema creation script.

**Approach:** Use Cloud SQL Proxy to create a local Unix socket connection, then run `python main.py` which calls `SQLModel.metadata.create_all(engine)`.

**Steps:**

1. **Download and install Cloud SQL Proxy** (if not already installed):
```bash
curl -o cloud-sql-proxy https://storage.googleapis.com/cloud-sql-connectors/cloud-sql-proxy/v2.21.1/cloud-sql-proxy.darwin.amd64
chmod +x cloud-sql-proxy
mv cloud-sql-proxy /usr/local/bin/
```

2. **Start Cloud SQL Proxy in background:**
```bash
cloud-sql-proxy {INSTANCE_CONNECTION_NAME} &
```

This creates a Unix socket at `/cloudsql/{INSTANCE_CONNECTION_NAME}`.

3. **Set DATABASE_URL to use Unix socket:**
```bash
export DATABASE_URL="postgresql+psycopg2://{SERVICE_ACCOUNT_EMAIL}:@/finance_tracker?host=/cloudsql/{INSTANCE_CONNECTION_NAME}"
```

Note: Password is empty (`:@`) for IAM authentication. IAM auth with Cloud SQL Proxy uses service account credentials from `gcloud auth application-default login`.

4. **Run schema creation:**
```bash
python main.py
```

This executes `SQLModel.metadata.create_all(engine)` which creates all tables (users, accounts, liabilities, account_types, liability_types, snapshots).

5. **Verify tables created:**
```bash
# Connect via proxy and list tables
gcloud sql connect finance-tracker-db --user={SERVICE_ACCOUNT_EMAIL} --database=finance_tracker
\dt
\q
```

6. **Stop Cloud SQL Proxy:**
```bash
killall cloud-sql-proxy
```

**If IAM auth fails:** Fall back to database password approach:
- Retrieve password from Cloud SQL instance creation
- Update DATABASE_URL: `postgresql+psycopg2://postgres:{PASSWORD}@/finance_tracker?host=/cloudsql/{INSTANCE_CONNECTION_NAME}`
- Re-run main.py
  </action>
  <verify>gcloud sql connect finance-tracker-db --user={SERVICE_ACCOUNT_EMAIL} --database=finance_tracker -e "\dt"</verify>
  <done>All tables (users, accounts, liabilities, account_types, liability_types, snapshots) exist in Cloud SQL production database</done>

<task type="auto">
  <name>Deploy to Cloud Run with configuration</name>
  <files>N/A - infrastructure task</files>
  <action>
Deploy the Docker image to Cloud Run with Cloud SQL connection, Secret Manager mount, and free tier constraints.

**Deploy command:**
```bash
gcloud run deploy finance-tracker \
  --image gcr.io/{PROJECT_ID}/finance-tracker:latest \
  --platform managed \
  --region us-central1 \
  --allow-unauthenticated \
  --max-instances=1 \
  --memory=512Mi \
  --cpu=1 \
  --timeout=300 \
  --add-cloudsql-instances={INSTANCE_CONNECTION_NAME} \
  --set-env-vars="DATABASE_URL=postgresql+psycopg2://{SERVICE_ACCOUNT_EMAIL}:@/finance_tracker?host=/cloudsql/{INSTANCE_CONNECTION_NAME}" \
  --set-secrets=/secrets/firebase.json=finance-tracker-firebase-creds:latest \
  --set-env-vars="FIREBASE_CREDENTIALS_PATH=/secrets/firebase.json" \
  --set-env-vars="FIREBASE_WEB_API_KEY={USER_FIREBASE_WEB_API_KEY}" \
  --set-env-vars="FIREBASE_AUTH_DOMAIN={USER_FIREBASE_AUTH_DOMAIN}" \
  --set-env-vars="FIREBASE_PROJECT_ID={USER_FIREBASE_PROJECT_ID}"
```

**Flag explanations:**
- `--allow-unauthenticated`: Public access (app handles auth internally via Firebase)
- `--max-instances=1`: Free tier constraint (per CONTEXT.md user decision)
- `--memory=512Mi`, `--cpu=1`: Minimum resources for Streamlit
- `--timeout=300`: 5-minute timeout for slow first loads
- `--add-cloudsql-instances`: Mounts Unix socket at `/cloudsql/{INSTANCE_CONNECTION_NAME}`
- `--set-env-vars DATABASE_URL`: Connection string with Unix socket path
- `--set-secrets`: Mounts Firebase credentials JSON as file (NOT env var)
- Firebase web config env vars: Required by frontend auth component

**Before executing:** Prompt user for Firebase web config values (API key, auth domain, project ID from Firebase Console).

**After deploy:** Capture Cloud Run service URL from output.
  </action>

  <done>Cloud Run service deployed, healthy, and accessible via HTTPS URL</done>
