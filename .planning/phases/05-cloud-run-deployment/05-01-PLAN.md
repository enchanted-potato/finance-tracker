<task type="auto">
  <name>Update Dockerfile CMD to honor Cloud Run PORT</name>
  <files>Dockerfile</files>
  <action>
Update the Dockerfile CMD line to use Cloud Run's PORT environment variable with fallback to 8501:

**Current CMD:**
```dockerfile
CMD ["uv", "run", "streamlit", "run", "frontend/main.py", "--server.port=8501", "--server.address=0.0.0.0"]
```

**Updated CMD (shell form for PORT expansion):**
```dockerfile
CMD streamlit run frontend/main.py --server.port=${PORT:-8501} --server.address=0.0.0.0
```

**Critical changes:**
- Remove exec form (`["uv", "run", ...]`) â€” switch to shell form for `${PORT:-8501}` expansion
- Use shell parameter expansion `${PORT:-8501}` â€” defaults to 8501 if PORT unset (local dev)
- Remove `uv run` wrapper â€” not needed, dependencies installed in image via `uv sync`
- Keep `--server.address=0.0.0.0` â€” required for container networking

**Why shell form:** Cloud Run sets PORT env var (default 8080 but can vary). Shell form allows variable expansion; exec form treats `${PORT}` as literal string. Per RESEARCH.md Pitfall 2, hardcoded port causes health check failures.

Leave all other Dockerfile lines unchanged (FROM, WORKDIR, uv install, COPY, ENV, EXPOSE).
  </action>
  <verify>grep -E 'CMD.*\$\{PORT' Dockerfile</verify>
  <done>Dockerfile CMD uses ${PORT:-8501} with shell form, streamlit honors Cloud Run's assigned port</done>
</task>
