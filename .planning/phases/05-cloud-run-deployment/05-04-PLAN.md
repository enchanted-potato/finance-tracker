  <action>Obtain Firebase UID from Firebase Console</action>
  <instructions>
The migration script requires the Firebase UID of the authenticated production user. Since the Cloud Run app is not yet deployed (migration happens BEFORE deployment per CONTEXT.md), obtain the UID from Firebase Console instead of production app login.

**Steps to obtain Firebase UID from Firebase Console:**

**Option A - Pre-authenticate user in Firebase Console:**

1. **Navigate to Firebase Console:**
   - Visit: https://console.firebase.google.com/
   - Select your project

2. **Enable Google Sign-In provider** (if not already enabled):
   - Navigate to: Authentication -> Sign-in method
   - Click "Google" provider
   - Enable and save

3. **Add test user to Firebase Authentication:**
   - Navigate to: Authentication -> Users
   - Click "Add user" (manual creation)
   - Email: Your Google account email
   - Auto-generate password (won't be used for Google Sign-In)
   - Click "Add user"

4. **Copy Firebase UID:**
   - The newly created user appears in the Users list
   - Copy the "User UID" column value (format: random alphanumeric string)

**Option B - Use Firebase Admin SDK to create user (Python):**

```bash
# Install firebase-admin if needed
pip install firebase-admin

# Create user via Firebase Admin SDK
python -c "
import firebase_admin
from firebase_admin import credentials, auth

# Initialize Firebase Admin SDK with service account
cred = credentials.Certificate('firebase-credentials.json')
firebase_admin.initialize_app(cred)

# Create user with email
email = input('Enter your Google account email: ')
user = auth.create_user(email=email)
print(f'Firebase UID: {user.uid}')
print(f'Email: {user.email}')
"
```

**Option C - Use existing Firebase user (if you've used this Firebase project before):**

1. Navigate to: Firebase Console -> Authentication -> Users
2. Find existing user with your Google account email
3. Copy the "User UID" value

**What you should receive:**
- Firebase UID (format: 28-character alphanumeric string, e.g., `kX9mN2pQ4rS1tU6vW8yZ0aB3cD5e`)
- Email matching your Google account

**Important:**
- Do NOT use 'test-user' — that's what we're migrating FROM
- This UID will be the production user after deployment
- The migration will transfer all test-user data to this UID

Execute the migration script to update all test-user data to the real Firebase UID obtained from Firebase Console.

**Steps:**

1. **Start Cloud SQL Proxy:**
```bash
cloud-sql-proxy {INSTANCE_CONNECTION_NAME} &
PROXY_PID=$!
```

2. **Set DATABASE_URL:**
```bash
export DATABASE_URL="postgresql+psycopg2://{SERVICE_ACCOUNT_EMAIL}:@/finance_tracker?host=/cloudsql/{INSTANCE_CONNECTION_NAME}"
```

3. **Run migration in dry-run mode first (verification):**
```bash
python scripts/migrate_test_user.py {FIREBASE_UID} --dry-run
```

Expected output:
```
DRY RUN - No changes will be made
Target UID: {FIREBASE_UID}
SQL statements that would be executed:
  account_types: UPDATE account_types SET user_id = :new_uid WHERE user_id = 'test-user'
  accounts: UPDATE accounts SET user_id = :new_uid WHERE user_id = 'test-user'
  liabilities: UPDATE liabilities SET user_id = :new_uid WHERE user_id = 'test-user'
  snapshots: UPDATE snapshots SET user_id = :new_uid WHERE user_id = 'test-user'
  users: DELETE FROM users WHERE id = 'test-user'
```

4. **Run actual migration:**
```bash
python scripts/migrate_test_user.py {FIREBASE_UID}
```

The script will prompt:
```
This will migrate all data from 'test-user' to '{FIREBASE_UID}'.
This operation cannot be undone.
Continue? [y/N]:
```

Type `y` and press Enter.

Expected output:
```
Starting migration from 'test-user' to '{FIREBASE_UID}'
  account_types: X rows affected
  accounts: X rows affected
  liabilities: X rows affected
  snapshots: X rows affected
  users: 1 rows affected
Migration completed successfully

Migration Summary:
  account_types: X rows
  accounts: X rows
  liabilities: X rows
  snapshots: X rows
  users: 1 rows
```

5. **Verify no test-user rows remain:**
```bash
# Query for any remaining test-user references
```

Expected output: All counts should be 0.


- Check Cloud SQL IAM auth is enabled
- Review error message for FK constraint violations or connection errors

**Note on timing:** This migration happens BEFORE Cloud Run deployment (Plan 03), honoring CONTEXT.md decision "migrate test-user data before first deploy". The Firebase UID is obtained from Firebase Console (not production app login) to make this possible.
