<task type="auto">
  <name>Simplify get_or_create_user function in auth_service.py</name>
  <files>app/services/auth_service.py</files>
  <action>
Simplify the get_or_create_user() function to only validate the UID and return it, since we no longer create User records.

**Replace the entire function with:**
```python
def get_or_create_user(
    session: Session, uid: str, email: str, display_name: str
) -> str:
    """Validate user ID and return it.

    Args:
        session: SQLModel database session (unused, kept for compatibility)
        uid: Firebase user ID
        email: User email (unused, kept for compatibility)
        display_name: User display name (unused, kept for compatibility)

    Returns:
        Validated Firebase UID string

    Raises:
        ValueError: If uid is 'test-user' (blocked in production)
    """
    # Block test-user from production
    if uid == 'test-user':
        raise ValueError("Invalid user_id: 'test-user' is not allowed in production")

    logger.info(f"Authenticated user: {uid} ({email})")
    return uid
```

**Why:**
- No users table means no User records to create or fetch
- Function now just validates the UID (blocks 'test-user') and returns it
- Signature kept compatible with existing callers (session, uid, email, display_name)
- Return type changed from User to str (the Firebase UID)
- Parameters kept for compatibility even though only uid is used

**Note:** This may require updating callers in frontend code to expect a string instead of User object. Check main.py for any references to user object properties.
  </action>
  <verify>grep -A 5 "def get_or_create_user" app/services/auth_service.py</verify>
  <done>get_or_create_user simplified to return string UID with test-user validation; verification shows new function signature</done>
</task>
