<task type="checkpoint:human-action" gate="blocking">
  <name>Obtain Firebase UID from production login</name>
  <action>Obtain Firebase UID from production login</action>
  <instructions>
The migration script requires the Firebase UID of the authenticated production user. This UID is only available after logging in to the deployed Cloud Run application.

**Steps:**

1. **Log in to production app:**
   - Visit the Cloud Run URL: `https://finance-tracker-XXXXX-uc.a.run.app` (from Plan 03 output)
   - Click "Sign in with Google"
   - Complete Google authentication

2. **Retrieve Firebase UID:**

   **Option A - Via Cloud Logging:**
   ```bash
   gcloud logging read "resource.type=cloud_run_revision AND resource.labels.service_name=finance-tracker AND textPayload=~'get_or_create_user'" --limit=5 --format=json
   ```
   Look for loguru log entry showing `get_or_create_user` with the UID.

   **Option B - Via database query:**
   ```bash
   # Start Cloud SQL Proxy
   cloud-sql-proxy {INSTANCE_CONNECTION_NAME} &

   # Connect to database
   gcloud sql connect finance-tracker-db --user={SERVICE_ACCOUNT_EMAIL} --database=finance_tracker

   # Query users table
   SELECT id, email FROM users ORDER BY created_at DESC LIMIT 5;
   \q

   # Stop proxy
   killall cloud-sql-proxy
   ```
   The most recent user row (not 'test-user') is the production Firebase UID.

   **Option C - Via Python (if gcloud sql connect fails):**
   ```bash
   # Start proxy
   cloud-sql-proxy {INSTANCE_CONNECTION_NAME} &

   # Run Python query
   DATABASE_URL="postgresql+psycopg2://{SERVICE_ACCOUNT_EMAIL}:@/finance_tracker?host=/cloudsql/{INSTANCE_CONNECTION_NAME}" \
     python -c "from app.database import engine; from sqlmodel import Session, select; from app.models import User; session = Session(engine); users = session.exec(select(User).order_by(User.created_at.desc()).limit(5)).all(); [print(f'{u.id} - {u.email}') for u in users]"

   # Stop proxy
   killall cloud-sql-proxy
   ```

3. **Record Firebase UID:**

   Copy the Firebase UID (format: random alphanumeric string, e.g., `abc123xyz456def789`)

**What you should see:**
- Two users in the database: 'test-user' and a Firebase UID (long alphanumeric string)
- The Firebase UID user email matches your Google account

**Important:** Do NOT use 'test-user' as the UID — that's what we're migrating FROM.
  </instructions>
  <resume-signal>Type "ready" and provide the Firebase UID: ____</resume-signal>
</task>

<task type="auto">
  <name>Run migration script via Cloud SQL Proxy</name>
  <action>
Execute the migration script to update all test-user data to the real Firebase UID.

**Steps:**

1. **Start Cloud SQL Proxy:**
```bash
cloud-sql-proxy {INSTANCE_CONNECTION_NAME} &
PROXY_PID=$!
```

2. **Set DATABASE_URL:**
```bash
export DATABASE_URL="postgresql+psycopg2://{SERVICE_ACCOUNT_EMAIL}:@/finance_tracker?host=/cloudsql/{INSTANCE_CONNECTION_NAME}"
```

3. **Run migration in dry-run mode first (verification):**
```bash
python scripts/migrate_test_user.py {FIREBASE_UID} --dry-run
```

Expected output:
```
DRY RUN - No changes will be made
Target UID: {FIREBASE_UID}
SQL statements that would be executed:
  account_types: UPDATE account_types SET user_id = :new_uid WHERE user_id = 'test-user'
  accounts: UPDATE accounts SET user_id = :new_uid WHERE user_id = 'test-user'
  liabilities: UPDATE liabilities SET user_id = :new_uid WHERE user_id = 'test-user'
  snapshots: UPDATE snapshots SET user_id = :new_uid WHERE user_id = 'test-user'
  users: DELETE FROM users WHERE id = 'test-user'
```

4. **Run actual migration:**
```bash
python scripts/migrate_test_user.py {FIREBASE_UID}
```

The script will prompt:
```
This will migrate all data from 'test-user' to '{FIREBASE_UID}'.
This operation cannot be undone.
Continue? [y/N]:
```

Type `y` and press Enter.

Expected output:
```
Starting migration from 'test-user' to '{FIREBASE_UID}'
  account_types: X rows affected
  accounts: X rows affected
  liabilities: X rows affected
  snapshots: X rows affected
  users: 1 rows affected
Migration completed successfully

Migration Summary:
  account_types: X rows
  accounts: X rows
  liabilities: X rows
  snapshots: X rows
  users: 1 rows
```

5. **Verify no test-user rows remain:**
```bash
# Query for any remaining test-user references
python -c "
from app.database import engine
from sqlmodel import Session, text
session = Session(engine)
tables = ['account_types', 'accounts', 'liabilities', 'snapshots', 'users']
for table in tables:
    result = session.exec(text(f\"SELECT COUNT(*) FROM {table} WHERE user_id = 'test-user'\"))
    count = result.one()
    print(f'{table}: {count} test-user rows')
session.close()
"
```

Expected output: All counts should be 0.

6. **Stop Cloud SQL Proxy:**
```bash
kill $PROXY_PID
```

**If migration fails:**
- Check DATABASE_URL is correct (Unix socket path)
- Verify Cloud SQL Proxy is running (`ps aux | grep cloud-sql-proxy`)
- Check Cloud SQL IAM auth is enabled
- Review error message for FK constraint violations or connection errors
